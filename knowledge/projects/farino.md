# 7. 核心项目二：法奥机器人智能客服平台

- 仓库：https://github.com/sleeping-Zack/farino
- 时间：2026.01—2026.06
- 性质：南昌大学 × 法奥机器人校企合作
- 角色：Agent 后端开发 / AI 应用开发
- 定位：企业场景和大型存量工程项目

## 项目描述

面向机器人技术咨询、资料检索与售后服务场景，基于 AIFlowy 开展企业级二次开发，构建包含智能问答、Agent 任务规划、QA 数据沉淀和售后工单管理的客服工作台，支持管理端、用户端及企业内网私有化部署。

## 技术栈

Java、Spring Boot、MyBatis-Flex、Vue 3、TypeScript、Element Plus、MySQL、Redis、Qwen、SSE、Docker Compose、Nginx、AIFlowy、Workflow、Plugin、MCP。

## 个人贡献

### 1. Direct / Agentic 两级路由

针对简单问题也进入完整 Agent 流程、增加响应时间和模型调用开销的问题，设计 Direct / Agentic 两级路由。

结合规则判断与 Qwen 意图识别，将普通寒暄、简单问答等请求直接交由对话模型处理；对于需要外部能力的复杂请求，再进入知识库、Workflow、插件及 MCP 工具编排链路，减少不必要的 Agent 执行步骤。

### 2. Agent 动态规划模块

实现 Agent 动态规划模块，根据用户问题和 Bot 已绑定的知识库、Workflow、插件及 MCP 能力生成 3—5 步执行计划。

在计划生成和执行过程中增加：

- 工具类型校验
- 工具 ID 合法性校验
- 参数结构与必填项校验
- 无效工具和异常结果降级处理

通过 SSE 向前端同步计划生成、步骤开始、工具执行、步骤完成和任务结束等状态，使用户能够查看 Agent 的任务执行过程。

### 3. QA 数据闭环模块

开发 QA 数据闭环模块，从历史会话中抽取 User–Assistant 问答对，并完成：

- 上下行消息配对
- 空内容和低质量内容过滤
- 基于消息 ID 的幂等去重
- 原始问答内容保留
- 人工编辑与审核
- Markdown 批量导出

支持将审核后的优质问答整理为知识库文档，形成“历史会话—问答筛选—人工审核—文档导出—知识库回流”的数据沉淀流程。

### 4. 售后工单流程

开发用户端与管理端售后工单功能，支持问题类型、优先级和以下五类状态流转：

- 待处理
- 处理中
- 待补充
- 已解决
- 已关闭

依据用户身份和所属关系进行工单数据隔离，用户端仅访问本人提交的工单，管理端负责查询、处理和更新工单状态。

### 5. 私有化部署与服务编排

使用 Docker Compose 编排后端、前端、MySQL 和 Redis 等服务，通过 Nginx 完成：

- API 反向代理
- 前端及静态资源访问
- SSE 长连接与超时配置
- 前后端服务转发

支持平台在企业内网环境中进行私有化部署和统一启动，参与处理服务依赖、环境配置和长连接中断等部署问题。

## 贡献边界

该项目是在 AIFlowy 现有平台基础上进行的企业级二次开发，个人贡献集中在 Agent 路由、动态规划、QA 数据闭环、售后工单和部署链路。

不表述为独立开发整个 AIFlowy，也不将 AIFlowy 原有的知识库、工作流、插件和平台基础能力描述为个人从零实现。

## 项目价值

该项目体现了候选人在真实企业客服场景下进行大型存量系统二次开发的能力，能够围绕 Agent 执行效率、任务规划、数据沉淀、业务流程和私有化部署完成前后端链路落地。
