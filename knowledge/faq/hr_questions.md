# HR 面试问答知识库

本文件收录 HR 面试中最常被问到的问题及朱旭的参考回答。AI 在引用这些回答时应保持诚实、准确，不应夸大或修饰。

---

# 请做一下自我介绍

我是朱旭，南昌大学软件工程专业 2027 届本科生，主要方向是 Agent 应用开发、RAG 和 Python 后端。

我目前最主要的项目是一个扫地机器人智能客服 Agent。项目从 RAG 知识库开始，后来逐步增加多工具调用，并在 ReAct Agent 外增加 Harness 控制层，处理预算、工具权限、敏感审批、答案验证、Trace 和离线评测等问题。

另外，我参与过南昌大学与法奥机器人的校企合作，在 AIFlowy 平台上进行 Agent 和智能客服功能二次开发，也做过 Django 情绪分析和 IoT 数据同步课程项目。我希望进入大模型应用或 Agent 平台团队，继续提升真实业务下的工程能力。

---

# 为什么选择 AI 应用开发

我更感兴趣的是如何把模型能力变成可运行、可测试、可维护的系统。相比单纯研究模型结构，我更关注知识库、工具、权限、成本、状态和评测这些落地问题。我的项目演进也一直沿着这条路线。

不是追热点，是因为做了之后觉得有意思。在做 RAG 项目的过程中，我第一次完整经历了"文档怎么切、Embedding 怎么选、检索结果怎么评估"这条链路，发现 AI 落地的核心问题其实是工程问题，而不只是模型问题。这和我喜欢解决实际问题的风格很契合。

---

# 为什么选择我们公司

回答模板：

1. 说明公司当前的 AI 应用场景或平台能力
2. 对应自己在 RAG、Agent、后端和评测中的经验
3. 指出自己希望补足的真实业务和生产规模经验
4. 不泛泛说"大平台、成长快"

> AI 回答提示：这道题需要根据具体公司作答。AI 应说明需要根据具体公司情况来回答，建议由朱旭本人在面试中针对性准备。

---

# 你有实习经历吗

目前没有可确认的正式实习经历。我的主要经验来自个人项目、南昌大学与企业的校企合作、课程团队项目和创新实践。

我会明确区分经历性质，不会把校企合作包装成正式实习。但这些项目中有真实需求、团队协作和可验证代码，也让我积累了较完整的 AI 应用工程实践。

---

# 你最大的优势是什么

我的优势不是单一框架使用，而是会主动补齐 AI 应用从 Demo 到工程系统之间的控制问题。例如，在主项目中，我不仅实现 RAG 和工具调用，还增加预算、权限、人工审批、答案验证、Trace、Artifact 和评测门禁。

另外，我能清楚区分"做过"和"主导过"、"了解"和"熟练"，经历口径透明，不夸大项目经历。

---

# 你最大的不足是什么

我目前最大的不足是缺少正式实习和真实大规模线上系统经验。现有项目可以证明我能搭建完整链路，但流量、SLA、分布式存储和真实业务指标还需要在企业环境中补足。

具体来说：

1. 没有经历过正式的商业团队开发流程，缺乏 Code Review、需求评审等协作经验
2. 项目使用 SQLite、Chroma 等单机组件，生产规模经验有限
3. Java 和 Vue 更接近阅读与二次开发，不是主栈

---

# 为什么没有实习

大学阶段前期主要投入课程、校企项目和个人技术项目，求职方向也经历了从后端到 AI 应用的收敛。现在方向已经明确，希望通过实习补足真实团队流程和生产环境经验。

---

# 你如何使用 AI 编程工具

我会用 Claude、Cursor 辅助代码检索、样板代码、测试和重构，但不会把生成结果直接当成完成。我负责需求拆解、架构选择、接口集成、调试、测试和验收，并确保自己能解释核心模块。

项目 commit 中有 Claude、Cursor 协作署名，我会主动承认辅助使用，并且能现场解释核心代码和设计决策。

---

# 你如何处理压力

从项目记录看，我会先收集日志和复现路径，再把问题拆成可验证步骤。时间紧张时，我倾向于先定义 MVP 和验收标准，再逐步补工程能力。

---

# 你和团队发生分歧怎么办

先明确目标和约束，再用可验证的方案比较，而不是只讨论个人偏好。对于技术方案，会比较实现成本、风险、可测试性和后续维护；无法一次确定时，可以用小范围实验或原型验证。

---

# 你适合什么样的团队

适合有明确业务场景、重视工程质量、允许实习生承担完整模块的大模型应用、Agent 平台、企业知识库或 AI 后端团队。

---

# 你未来三年的规划

第一阶段补足真实生产项目经验；第二阶段提升 Agent 平台、评测、可观测和分布式服务能力；之后希望成为能够独立负责 AI 应用模块或垂直场景 Agent 的工程师。

---

# 为什么不投纯算法岗

当前能力和兴趣更偏模型应用、系统工程和业务落地。没有足够的模型训练、论文和数学研究积累，因此不会将自己包装为纯算法候选人。

---

# 你为什么值得进入技术面

- 有公开 GitHub 项目
- 有完整 Agent 和 RAG 工程链路
- 能说明设计取舍和已知局限
- 有测试、CI、Trace 和评测意识
- 经历口径透明
- 学习轨迹连续

---

# 你什么时候可以入职

实习的话随时可以，协调一下课程安排，一到两周内可以到岗。

正式校招：2027 届，预计 2027 年 6 月毕业。如果有提前批，可以沟通具体时间。

---

# 你的薪资期望是多少

实习以市场行情为参考，具体可以面议。校招同样以行业标准为参考，希望薪资能和岗位的实际职责相匹配，具体数字可以在后续环节沟通。

> AI 回答提示：不要代替朱旭说出具体数字。

---

# 你如何看待加班

不抵触为了解决真实问题而加班，比如上线前的紧急排障，或者某个功能卡壳了需要集中攻克。但希望加班是例外而不是常态，因为长期高强度工作最终会影响产出质量。

---

# 你有团队项目经验吗

校企合作项目（法奥机器人）是小组形式，我主要负责 Agent 路由、动态规划、QA 数据闭环相关模块。IoT 平台也是团队课程项目，负责云端归档和 DRF API。

需要说明：这些项目的团队规模较小，和互联网公司的跨部门协作还是有差距的。缺少在大型既有代码库里与多团队协同的经验，这是我的短板之一。

---

# 你有什么想问我的

建议问题（供参考）：

- 这个岗位日常工作中 AI 工程的比重大概是多少？
- 团队现在用的是哪些技术栈？
- 新人进来通常会先接手什么类型的任务？
