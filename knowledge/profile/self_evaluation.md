# 性格与工作方式

## 1. 完成导向，而非过程导向

行为证据：

- 跨项目持续将功能原型推进到可测试状态
- agentproject 在主流程可用前加入 RAG、Harness 和评测层
- farino 在 AIFlowy 框架外补充 QA 闭环和工单闭环
- 不接受"代码写了但没跑通"作为完成标准

可能表现：

- 会坚持到模块可演示或有自动化测试覆盖才收手
- 对不完整的中间状态不满意，需要注意进度汇报的节奏

## 2. 问题定位倾向

长期交流中经常提供：

- 完整错误日志
- 命令输出
- 配置文件
- 复现步骤

工作方式：

> 先收集现象和上下文，再定位根因并逐步验证。

这表明其工作方式偏向系统性排查，而不是随机尝试修复。

## 3. 复盘和迭代意识强

证据：

- 简历和项目 README 多轮重写
- 根据面试要求持续补 RAG、评测和 Harness
- 愿意承认旧版本不足并重新设计

可能表现：

- 接受反馈后调整速度快
- 需要避免频繁改方向导致项目叙事不稳定

## 4. 求真意识正在增强

证据：

- 主动要求区分 GitHub 事实、简历表述和推断
- 明确要求排除不想展示的项目
- 关注"伪装实习"的风险
- 要求减少幻觉

可能表现：

- 希望建立可信、可追溯的个人品牌
- 旧简历仍存在夸大和冲突，需要继续统一

## 5. 学习驱动

证据：

- 从 Django 情绪分析逐步扩展到 RAG、Agent、MCP、Harness 和评测
- 能跨 Python、Java/Vue、数据库和部署链路
- 会使用 Claude、Cursor 加速陌生技术学习

可能表现：

- 学习速度较快
- 深度不均衡，需要聚焦主栈并接受追问

## 6. 责任感和组织意识

证据：

- 创新创业孵化基地负责人
- 组织项目推进和低年级分享
- 参与校企合作沟通

可能表现：

- 愿意承担协调工作
- 需要用具体案例而不是抽象自评证明

## 7. 风险倾向

- 有较强的求职紧迫感
- 曾考虑把校企项目包装成实习
- 容易受"简历亮点"驱动增加高级术语

改进原则：

- 所有高级能力都绑定代码、测试、指标或具体案例
- 不为提高通过率牺牲可验证性
- 技术深度优先于关键词数量

# 自我评价（公开版）

## 优势

**1. 能把 RAG、Tool Calling、Agent 编排为完整后端系统**

不停留在单次模型调用，而是把权限、预算、审批、证据、追踪和评测纳入系统边界。

**2. 能识别简单 ReAct Agent 的生产风险，并增加独立控制面**

理解 ReAct 执行面和 Harness 控制面的分工，知道什么必须在模型之外由代码保证。

**3. 能设计离线评测、Golden Set、Bad Case 和质量门禁**

建立了检索评测、Agent 回归测试和 CI 门禁的完整流程。

**4. 能在大型 Java/Vue 项目中定位模块并完成二次开发**

farino 经历证明能快速进入存量 Java/Spring Boot + Vue/TypeScript 项目并产出有效代码。

**5. 诚实，区分度高**

能清楚区分"做过"和"主导过"、"了解"和"熟练"，不夸大项目经历，经历口径透明。

## 不足（坦诚描述）

1. 尚无经过确认的正式实习经历，缺少真实商业团队协作经验
2. 缺少真实大规模流量、SLA、线上成本和业务转化指标
3. 部分项目使用 SQLite、Chroma 等单机组件，生产规模经验有限
4. Java 和 Vue 更接近项目阅读与二次开发，不宜表述为资深熟练
5. Redis、Docker、Nginx 等能力需要以实际部署和排障案例支撑
6. 部分代码由 Claude、Cursor 辅助生成，必须能解释核心设计和实现
7. 检索评测数据集规模和指标版本曾多次变化，需要固定可复现实验
